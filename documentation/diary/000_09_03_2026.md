# NORA Development Diary

## Entry 000 — 09/03/2026

### Context

NORA is a personal long-term project whose objective is the development of a humanoid system composed of both hardware and software components. The idea is to create a physical platform capable of hosting multiple AI agents working together as part of a unified system, effectively behaving as a "person-like" entity from the perspective of interaction and autonomy.

The project began at the end of 2024. During the first months some initial work was done on both the frontend and backend of the system. However, due to lack of time and other priorities, development stopped around the beginning of 2025.

More than one year later, on March 9th 2026, the project has been resumed. Only a portion of the original codebase has been recovered: part of the frontend and part of the backend. Much of the earlier work has been lost or intentionally discarded. This is not considered a problem because the technical knowledge and experience acquired since then would likely lead to a different and improved implementation anyway.

### Purpose of the Diary

One of the main difficulties encountered when returning to the project was the loss of context regarding previous design decisions, ideas, and experiments. Even when the code remains available, the reasoning behind it often disappears over time.

For this reason a development diary is introduced inside the repository under:

```
documents/diary/
```

The goal of this diary is to maintain continuity in the project by documenting development sessions. A new entry will be created each day the project is worked on. These entries will describe what was done, problems encountered, decisions taken, and possible next steps.

### Repository Documentation Structure

The repository now contains the following documentation structure:

```
documents/
    system/
    learnings/
    diary/
```

Where:

* **system** contains documentation about the architecture and structure of the NORA system.
* **learnings** contains technical knowledge acquired while developing the project.
* **diary** records the chronological development process of the project.

### Current State of the Project

At the moment of writing this entry the following actions have been completed:

* Recovery of part of the original frontend and backend code.
* Installation and configuration of the backend environment.
* Resolution of several dependency issues required to run the backend (Django, DRF, CORS headers, JWT authentication, Channels).
* Initialization of the database and migration execution.
* Debugging of the authentication system, including the resolution of duplicated user records that were causing login errors.

After solving these issues the backend can now run correctly in the local development environment.

### Development Philosophy Going Forward

This project is not intended as a commercial product or company project. Its purpose is learning, experimentation, and exploration of complex system design involving robotics, AI agents, and distributed software components.

Because of that, development will not be rushed. Instead, progress will occur gradually whenever time is available. The focus will be on maintaining continuity and documenting decisions so that the project remains understandable even after long pauses.

The development diary is expected to play a central role in maintaining this continuity.

### Next Steps

Immediate next steps include:

* Uploading the current state of the project to GitHub.
* Ensuring the backend and frontend environments can be reproduced easily.
* Continuing the development of the system architecture documentation.
* Gradually rebuilding the missing parts of the project with improved design decisions.
