# NORA Development Diary

This directory contains the development diary of the NORA project.

The purpose of this diary is to maintain continuity in the project by recording
development sessions, technical decisions, encountered problems, and progress
over time.

Each entry corresponds to a day in which work was performed on the project.

File naming convention:

XXX_DD_MM_YYYY.md

Where:
- XXX is the entry number
- DD_MM_YYYY is the date of the work session

## Notes

Not every day will necessarily have an entry.  
An entry is created only when actual development work is performed.

The goal of the diary is not to log time, but to preserve **context and decisions**
so that the project can be resumed easily even after long pauses.
